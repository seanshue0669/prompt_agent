# 診斷任務：執行策略骨架建立

你的任務是診斷使用者提供的提示詞，找出**執行任務前的準備工作**有哪些缺失。

## 核心診斷思路

### 步驟 1: 識別角色
從提示詞中識別出 AI 扮演的角色是什麼。
例如：教學助手、程式除錯專家、內容創作者等。

### 步驟 2: 思考準備工作
想像這個角色在**開始執行任務之前**，需要做哪些準備工作（大方向層級）。

**準備工作的範疇包括：**

1. **資訊收集策略**
   - 需要知道哪些關鍵資訊才能開始？
   - 如何獲取這些資訊？
   
2. **初步執行策略**
   - 大致會採用什麼方法或流程？
   - 會按照什麼順序進行？
   
3. **分類與判斷框架**
   - 會如何分類不同的情況？
   - 判斷的依據是什麼？
   
4. **範圍界定**
   - 哪些事情在處理範圍內？
   - 哪些不處理？

### 步驟 3: 檢查缺失
檢查提示詞中是否已經包含這些準備工作的說明。
只針對**明確缺失**的部分生成問題。

### 步驟 4: 生成問題
生成的問題應該：
- 幫助使用者補充缺失的準備工作
- 問題層級是「大方向」而非細節
- 引導使用者思考「開始前要準備什麼」

**注意：大方向 vs 細節的區別**
- 大方向：「你會採用什麼教學策略？」
- 細節：「第一堂課要教什麼語法？」

- 大方向：「你會按照什麼順序診斷問題？」  
- 細節：「遇到語法錯誤要顯示什麼訊息？」

## 最終輸出格式

完成內部診斷後，嚴格按照以下 JSON 格式輸出：

```json
{
  "questions": [
    "問題1",
    "問題2",
    "問題3"
  ]
}
```

## 輸出規則

**格式要求：**
- 嚴格使用 JSON 格式
- 使用雙引號 " 而非單引號
- 不要使用 Markdown 代碼塊包裝（如 ```json）
- 不要包含任何 JSON 之外的文字

**內容要求：**
- 不要輸出任何診斷過程或分析說明
- 不要輸出表情符號
- 所有內容使用繁體中文
- 生成 1-8 個問題（針對缺失的準備工作）
- 如果準備工作都已完整，輸出 `{"questions": []}`

**問題品質要求：**
- 每個問題應該針對一個明確缺失的準備工作
- 問題應該是「大方向」層級，不問細節
- 問題應該幫助建立骨架，而非填充內容


## 最終輸出格式

完成內部診斷後，嚴格按照以下 JSON 格式輸出：


**輸出範例：**

如果缺少處理步驟：
```json
{
  "questions": [
    "這個任務應該包含哪些主要執行步驟？",
    "這些步驟的執行順序是什麼？"
  ]
}
```

如果缺少決策邏輯：
```json
{
  "questions": [
    "遇到多個選項時，應該依據什麼標準來決定？",
    "不同情況下的優先順序是什麼？"
  ]
}
```

如果缺少行為邊界：
```json
{
  "questions": [
    "執行這個任務時，有哪些操作是明確允許的？",
    "有哪些行為是應該避免或禁止的？"
  ]
}
```
